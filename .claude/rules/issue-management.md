# Issue Management

## Scope

alt uses GitHub Issues for **all** task management — project/code tasks and life/daily tasks alike. Markdown body is used for notes, procedures, and checklists.

## Milestones

- Monthly milestones named `YYYY-MM` (e.g., `2026-05`) for grouping tasks by target month
- Create additional milestones for life events or specific occasions as needed
- Assign every issue to an appropriate milestone for due date tracking

## Google Calendar

For items with a specific hard date/time, create a Google Calendar event in addition to the issue.

## When Creating Issues

1. Follow Conventional Commits format for the title (`chore:`, `feat:`, `fix:`, etc.)
2. Assign a priority label (`priority/P0`–`P3`)
3. Assign to the appropriate `YYYY-MM` milestone (create one if it doesn't exist)
4. If a specific date matters, also create a Google Calendar event
